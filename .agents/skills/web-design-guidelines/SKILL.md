# มาตรฐานคู่มือการออกแบบเว็บ (Web Interface Guidelines)

ทบทวนไฟล์โค้ดเพื่อให้มั่นใจว่าสอดคล้องกับมาตรฐานคู่มือการออกแบบเว็บ (Web Interface Guidelines)

## เมื่อใดควรใช้สกิลนี้
- เมื่อผู้ใช้ร้องขอให้ "รีวิว UI", "ตรวจสอบความสามารถในการเข้าถึง (Accessibility)", "ประเมินดีไซน์ (Audit Design)", "รีวิวประสบการณ์ผู้ใช้ (Review UX)" หรือ "ตรวจสอบเว็บไซต์เทียบกับหลักปฏิบัติที่ดีที่สุด"
- เมื่อต้องการตรวจสอบความถูกต้องของเลย์เอาต์หน้าจอและองค์ประกอบเว็บเทียบกับมาตรฐานสากล

## แหล่งข้อมูลมาตรฐาน (Guidelines Source)

ให้ดึงคู่มือเวอร์ชันใหม่ล่าสุดก่อนที่จะเริ่มการทบทวนเสมอ:

```
https://raw.githubusercontent.com/vercel-labs/web-interface-guidelines/main/command.md
```

ใช้เครื่องมือดึงข้อมูลเพื่อดึงข้อมูลกฎเกณฑ์ล่าสุด เนื้อหาที่ดึงมาจะประกอบไปด้วยกฎทั้งหมดและคำแนะนำสำหรับรูปแบบการรายงานผล

## ขั้นตอนการทำงาน (Usage)

เมื่อผู้ใช้ระบุไฟล์หรือรูปแบบแพทเทิร์น (เช่น `*.tsx`):
1. **ดึงข้อมูล (Fetch)** กฎเกณฑ์และคู่มือล่าสุดจาก URL ด้านบน
2. **อ่าน (Read)** ไฟล์ที่ระบุไว้
3. **ตรวจสอบ (Check)** องค์ประกอบของโค้ดโดยอิงจากกฎทุกข้อที่ดึงมา
4. **รายงานผล (Output)** การค้นพบโดยใช้รูปแบบที่ระบุไว้ในคู่มือที่ดึงมา (เช่น รูปแบบสั้นๆ อย่าง `file:line` format)

หากไม่ได้ระบุไฟล์เป้าหมาย ให้ถามผู้ใช้ว่าต้องการทบทวนไฟล์ไหน

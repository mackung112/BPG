# ตัวจัดการเนื้อหา Learning Portal (Learning Portal Manager)

ใช้สกิลนี้เมื่อผู้ใช้ (คุณครู) ต้องการเพิ่มวิชาใหม่ เพิ่มบทเรียน หรืออัปโหลดไฟล์เนื้อหาลงในเว็บไซต์ Learning Portal

## 📍 โครงสร้างระบบ (System Architecture)
เว็บไซต์นี้ใช้ระบบอ่านข้อมูลจากไฟล์ JSON เพื่อแสดงผลบนหน้าเว็บโดยอัตโนมัติ 
- **ฐานข้อมูลวิชา:** `src/data/curriculum.json`
- **ที่เก็บไฟล์เนื้อหา (Simulator):** `public/simulators/<รหัสวิชา>/<เลขหัวข้อ-ชื่อ>/`

## 🛠️ คำสั่งการทำงาน (Instructions)

เมื่อผู้ใช้สั่งให้ "เพิ่มวิชา" หรือ "เพิ่มเนื้อหา" ให้ปฏิบัติตามขั้นตอนต่อไปนี้:

### 1. การจัดการไฟล์เนื้อหา (Simulator / HTML Files)
หากผู้ใช้มีเนื้อหา HTML/CSS/JS หรือระบุโครงสร้างเนื้อหามาให้:
1. ให้สร้างโฟลเดอร์แยกตาม **รายวิชา** และแบ่งโฟลเดอร์ย่อยตาม **ลำดับหัวข้อ (เช่น 1.1, 1.2, 1.3)** เพื่อง่ายต่อการไล่แก้ไข เช่น `public/simulators/programming-21902/1.1-intro/`
2. สร้างไฟล์ `index.html` (และไฟล์อื่นๆ ที่เกี่ยวข้อง) ไว้ในโฟลเดอร์หัวข้อย่อยนั้น
3. **ข้อควรระวัง:** ไฟล์ HTML ต้องสมบูรณ์และสามารถเปิดทำงานแบบเดี่ยวๆ ได้ (Standalone)

### 2. การอัปเดตไฟล์ฐานข้อมูล (curriculum.json)
ให้แก้ไขไฟล์ `src/data/curriculum.json` อย่างระมัดระวัง โครงสร้างของ JSON แบ่งตามระดับความลึกดังนี้:
`categories` (ปวช/ปวส) ➡️ `subjects` (วิชา) ➡️ `chapters` (บทเรียน/Unit) ➡️ `lessons` (หัวข้อย่อย)

**รูปแบบข้อมูล (Schema):**
```json
{
  "id": "รหัสภาษาอังกฤษ ห้ามซ้ำ",
  "name": "ชื่อวิชา หรือ ชื่อบท",
  "description": "คำอธิบายสั้นๆ (ถ้ามี)",
  "type": "simulator", // (เฉพาะ Lesson เท่านั้น)
  "contentUrl": "/simulators/<รหัสวิชา>/<เลขหัวข้อ-ชื่อ>/index.html" // (เฉพาะ Lesson เท่านั้น)
}
```

### 3. การเพิ่มวิชาจากแผนการสอน (Lesson Plans)
หากผู้ใช้อ้างอิงถึงไฟล์ในโฟลเดอร์ `lesson-plans/` (เช่น `programming-21902-2001.md`):
1. ให้อ่านโครงสร้าง Unit และหัวข้อหลักจากไฟล์ `.md` นั้น
2. แปลง Unit เป็น `chapters` 
3. แปลงหัวข้อหลัก (เช่น 1.1, 1.2) เป็น `lessons`
4. ลิงก์ `contentUrl` ให้ตรงกับที่ระบุในไฟล์ `.md` หากไม่มีระบุ ให้ตั้งเป็น `/simulators/placeholder.html` ไปก่อน

## ⚠️ ข้อควรระวัง (Warnings)
- ห้ามแก้ไขไฟล์โค้ด `.jsx` นอกเสียจากว่าผู้ใช้ร้องขอให้ปรับปรุงดีไซน์
- โครงสร้าง JSON จะต้องถูกต้องตามหลัก (Valid JSON) ทุกครั้งก่อนเซฟ เพื่อป้องกันเว็บพัง

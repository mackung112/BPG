# มาตรฐานโครงสร้างไฟล์หลักสูตร (Curriculum Standard)

เมื่อผู้ใช้ต้องการให้สร้างหรือแก้ไขไฟล์หลักสูตร (เช่น `docs/curriculum/py_curriculum.md`) จะต้องยึดตามรูปแบบและมาตรฐานด้านล่างนี้อย่างเคร่งครัด เพื่อให้ระบบนำไปสร้างเป็นข้อมูลในแอพพลิเคชัน (data.js) ได้อย่างมีประสิทธิภาพ

---

## 📄 กฎการตั้งชื่อไฟล์
- เก็บไว้ในโฟลเดอร์ `docs/curriculum/`
- ใช้ชื่อย่อของวิชาตามด้วย `_curriculum.md` (เช่น `py_curriculum.md`, `sql_curriculum.md`, `it_curriculum.md`)

---

## 🏗️ โครงสร้างเนื้อหา (Markdown Format)

### 1. ส่วนหัว (Header)
ต้องใช้ Heading 1 (`#`) ระบุชื่อวิชา พร้อมรหัสวิชาในวงเล็บ และสามารถใส่หมายเหตุสถานะได้
```markdown
# ชื่อวิชาภาษาไทย (รหัสวิชา) (สถานะถ้ามี เช่น ทำจบที่บทที่ X)
```
*ตัวอย่าง: `# หลักการเขียนโปรแกรมภาษาคอมพิวเตอร์ (21902-2001) (ทำจบที่บทที่ 6)`*

### 2. หน่วยการเรียนรู้ (Unit)
ต้องใช้ Heading 2 (`##`) นำหน้าด้วยคำว่า `Unit X:` เสมอ
```markdown
## Unit 1: ชื่อหน่วยการเรียนรู้
```

### 3. บทเรียน (Lesson)
ต้องใช้ Bullet point แบบขีด (`-`) ตามด้วยหมายเลข `X.Y` และชื่อบทเรียน
```markdown
- 1.1 ชื่อบทเรียน
```

### 4. รายละเอียดบทเรียน / หัวข้อย่อย (Sub-topic or Description)
ให้ความยืดหยุ่นภายใต้บทเรียน โดยสามารถเลือกเขียนได้ 2 รูปแบบตามความเหมาะสม เพื่อให้ AI นำไปสร้างเนื้อหาได้สะดวก:
**แบบที่ 1**: เป็นหัวข้อย่อยแบบระบุตัวเลข (เหมาะกับเนื้อหาที่เป็นทางการ มีโครงสร้างชัดเจน)
```markdown
  - 1.1.1 ชื่อหัวข้อย่อยแรก
  - 1.1.2 ชื่อหัวข้อย่อยที่สอง
```
**แบบที่ 2**: เป็นข้อความอธิบายขอบเขตเนื้อหา (เหมาะกับการบอกภาพรวมให้ AI สร้างเนื้อหาต่อ)
```markdown
  - อธิบายความหมายและองค์ประกอบของภาษาคอมพิวเตอร์
  - ยกตัวอย่างภาษาที่นิยมในปัจจุบัน
```

## 📝 ตัวอย่างโค้ดมาตรฐาน

```markdown
# การจัดการฐานข้อมูล (20411-2004)

## Unit 1: ระบบฐานข้อมูลเบื้องต้น

- 1.1 ความรู้พื้นฐานเกี่ยวกับระบบฐานข้อมูล
  - 1.1.1 นิยามและความหมายของข้อมูลและฐานข้อมูล
  - 1.1.2 ประวัติความเป็นมาของระบบฐานข้อมูล
  - 1.1.3 ความสำคัญและประโยชน์ของระบบฐานข้อมูล
- 1.2 สถาปัตยกรรมระบบฐานข้อมูล
  - อธิบายความแตกต่างของสถาปัตยกรรมทั้ง 3 ระดับ
  - ยกตัวอย่างความเป็นอิสระของข้อมูลแต่ละระดับให้เข้าใจง่าย

## Unit 2: แบบจำลองความสัมพันธ์ของข้อมูล (E-R Model)

- 2.1 แนวคิดพื้นฐานของ E-R Model
  - แนะนำให้รู้จัก Entity และ Attribute พร้อมยกตัวอย่างประกอบการออกแบบฐานข้อมูลจริง
```

---

## ⚠️ ข้อควรระวัง (Strict Rules)
1. **ลำดับตัวเลขต้องถูกต้อง**: ห้ามข้ามลำดับในระดับ Unit และ Lesson
2. **ห้ามใช้ Markdown อื่นแทรก**: หลีกเลี่ยงการใส่ตาราง รูปภาพ หรือโค้ดบล็อกในไฟล์ Curriculum
3. **การใช้งานภายใต้ Lesson**: สามารถเขียนระบุตัวเลข `X.Y.Z` หรือข้อความบรรยายภาพรวมเนื้อหาได้ตามความเหมาะสม

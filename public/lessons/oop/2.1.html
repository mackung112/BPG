4 เสาหลักของการออกแบบ (4 Pillars)

หัวใจสำคัญที่ทำให้การเขียนโค้ดแบบ OOP ทรงพลัง ปลอดภัย และนำกลับมาใช้ใหม่ได้ง่าย

💊

1. Encapsulation (การซ่อนข้อมูล)

การห่อหุ้มข้อมูลสำคัญไว้ข้างใน เหมือน เครื่องยนต์รถ ที่ถูกซ่อนไว้ใต้ฝากระโปรง คุณแค่เหยียบคันเร่ง (Method) รถก็วิ่ง โดยไม่ต้องเข้าไปยุ่งกับกลไกภายใน ช่วยป้องกันข้อผิดพลาด

🧬

2. Inheritance (การสืบทอด)

การนำคลาสเดิมมาสร้างคลาสใหม่พร้อมฟีเจอร์เพิ่ม เหมือนสร้างแม่พิมพ์ "รถสปอร์ต" ที่สืบทอดฟังก์ชันพื้นฐานมาจาก "รถยนต์ทั่วไป" แต่เติมระบบเทอร์โบเข้าไป ลดการเขียนโค้ดซ้ำซ้อน

🎭

3. Polymorphism (การหลายรูปแบบ)

คำสั่งเดียวกัน แต่อ็อบเจกต์ตอบสนองไม่เหมือนกัน เช่น สั่ง "เร่งเครื่อง" รถทั่วไปอาจจะค่อยๆ ไป แต่ถ้ารถสปอร์ตจะพุ่งตัวแรง (ดูได้ใน Simulator ด้านล่าง!) ชื่อคำสั่งเหมือนกันแต่ผลลัพธ์ต่างกัน

🎛️

4. Abstraction (การนามธรรม)

การแสดงให้ผู้ใช้เห็นเฉพาะส่วนที่จำเป็นต้องใช้งาน เหมือน พวงมาลัยและเบรก คุณแค่รู้ว่าหมุนซ้ายคือเลี้ยวซ้าย โดยไม่ต้องเข้าใจวงจรไฟฟ้าสลับซับซ้อนที่ควบคุมล้ออยู่เบื้องหลัง

หน้าตาโค้ดภาษา Python จริง 💻

main.py
# 1. การสร้าง Class (Blueprint) class Car: # Method กำหนด Attribute เริ่มต้น def __init__(self, brand, color): self.brand = brand # Attribute: ยี่ห้อ self.color = color # Attribute: สี self.speed = 0 # Attribute: ความเร็ว # Method สำหรับทำงาน (Action) def accelerate(self): self.speed += 10 print(f"รถ {self.brand} เร่งไปที่ {self.speed} กม./ชม.") # 2. การสร้าง Object ตัวเป็นๆ my_car = Car("Honda", "Red") # 3. เรียกใช้งาน Method ให้อ็อบเจกต์ขับเคลื่อน my_car.accelerate()

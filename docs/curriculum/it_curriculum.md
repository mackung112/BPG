# วิชานี้จะเป็นปฏิบัติ หลักสูตร: ระบบปฏิบัติการ บำรุงรักษา และเครือข่ายคอมพิวเตอร์เบื้องต้น

(Integrated IT: 31910-0001 & 31910-0005)

## 💻 บทที่ 1: ปูพื้นฐานระบบคอมพิวเตอร์และเครือข่าย (Introduction to Computer & Network Systems)

- 1.1 สถาปัตยกรรมคอมพิวเตอร์เบื้องต้น: ทำความรู้จัก ฮาร์ดแวร์ ซอฟต์แวร์ และวงจรการประมวลผลข้อมูล (Input, Process, Storage, Output)
- 1.2 ก้าวแรกสู่ระบบเครือข่าย: ความหมาย ประโยชน์ และองค์ประกอบพื้นฐานของการสื่อสารข้อมูล
- 1.3 ประเภทและโครงสร้างเครือข่าย (Network Types): ทำความรู้จัก LAN, MAN, WAN และรูปแบบ Client-Server vs Peer-to-Peer
- 1.4 โทโพโลยีเครือข่าย (Network Topology): รูปแบบการเชื่อมต่อ (Bus, Star, Ring, Mesh) และการเลือกใช้งานให้เหมาะสม
  🛠️ กิจกรรมปฏิบัติการ: การจัดทำบัญชีฮาร์ดแวร์สำหรับสำนักงาน (IT Asset Inventory)

## 🛠️ บทที่ 2: เจาะลึกฮาร์ดแวร์และการประกอบคอมพิวเตอร์ (Hardware & PC Assembly)

- 2.1 รู้จักชิ้นส่วนหลัก: CPU, Motherboard, RAM, Storage (HDD/SSD), PSU และ VGA
- 2.2 ความปลอดภัยในการปฏิบัติงาน: เครื่องมือช่างเบื้องต้น และการป้องกันอันตรายจากไฟฟ้าสถิต (ESD)
- 2.3 การประกอบคอมพิวเตอร์ทีละขั้นตอน: ตั้งแต่การเตรียมเคส ติดตั้งชิ้นส่วน ไปจนถึงการจัดสายไฟ (Cable Management)
- 2.4 การตรวจสอบระบบ (POST & BIOS): การทดสอบเปิดเครื่อง การอ่านรหัสแจ้งเตือน และการตั้งค่า BIOS/UEFI เบื้องต้น
  🛠️ กิจกรรมปฏิบัติการ: ถอดประกอบคอมพิวเตอร์ และทดสอบระบบ POST

## 💿 บทที่ 3: ระบบปฏิบัติการและการเตรียมความพร้อม (OS Installation & Deployment)

- 3.1 การติดตั้งระบบปฏิบัติการ (Operating System: OS)
  - 3.1.1 หน้าที่ของระบบปฏิบัติการ (OS) คือ ซอฟต์แวร์หลักที่ทำหน้าที่เป็นสื่อกลางระหว่างฮาร์ดแวร์และผู้ใช้ มีหน้าที่จัดสรรทรัพยากร ควบคุมการประมวลผล จัดการไฟล์ และรันโปรแกรม โดยมีการเตรียมความพร้อม ก่อนติดตั้งหรือใช้งานอุปกรณ์
  - 3.1.2 ประเภทของระบบปฏิบัติการ ระบบปฏิบัติการหลักที่ใช้ในปัจจุบัน แบ่งได้ 3 ประเภท คือ Windows, macOS และ Linux
- 3.2 การเตรียมความพร้อมก่อนติดตั้ง
  - 3.2.1 การสำรองข้อมูล (Backup)
  - 3.2.2 เช็คสเปกคอมพิวเตอร์
  - 3.2.3 การสร้างตัวติดตั้ง (Bootable USB) ด้วย Rufus
- 3.3 ขั้นตอนการติดตั้งระบบปฏิบัติการ (OS) Windows 10/11
- 3.4 การตั้งค่าหลังติดตั้งเสร็จ (Post-Installation)

  🛠️ กิจกรรมปฏิบัติการ: การทำ USB Bootable, ติดตั้ง OS และอัปเดตไดรเวอร์

## 🔌 บทที่ 4: สื่อกลางและการเชื่อมต่อทางกายภาพ (Network Media & Cabling)

4.1 อุปกรณ์เครือข่ายคอมพิวเตอร์พื้นฐาน (Basic Network Equipment)
อุปกรณ์เครือข่ายทำหน้าที่รับส่ง จัดเส้นทาง และประมวลผลสัญญาณข้อมูลคอมพิวเตอร์ในระบบเน็ตเวิร์ก มีอุปกรณ์หลักที่ควรรู้จักดังนี้:

สวิตช์เครือข่าย (Switch): ทำหน้าที่เป็นตัวกลางเชื่อมต่ออุปกรณ์คอมพิวเตอร์หลายเครื่องในวงแลน (LAN) เดียวกัน มีจุดเด่นคือสามารถส่งข้อมูลแยกเฉพาะไปยังพอร์ตของเครื่องปลายทางได้อย่างถูกต้องและรวดเร็ว

เราเตอร์ (Router): อุปกรณ์จัดเส้นทางข้อมูลระหว่างเครือข่าย ทำหน้าที่เชื่อมโยงวงแลนย่อยต่าง ๆ เข้าด้วยกัน และเป็นประตูทางออก (Gateway) เพื่อนำข้อมูลเชื่อมต่อไปยังเครือข่ายอินเทอร์เน็ตภายนอก

จุดเชื่อมต่อสัญญาณไร้สาย (Access Point): ทำหน้าที่กระจายสัญญาณไวไฟ (Wi-Fi) เพื่อให้อุปกรณ์พกพา เช่น สมาร์ตโฟน หรือแล็ปท็อป สามารถเชื่อมต่อเข้ามาใช้งานในวงเน็ตเวิร์กสายหลักได้

การ์ดเครือข่าย (Network Interface Card - NIC): แผงวงจรควบคุมหลักที่อยู่บนเมนบอร์ดของคอมพิวเตอร์ ทำหน้าที่แปลงข้อมูลดิจิทัลให้อยู่ในรูปของสัญญาณไฟฟ้าเพื่อรับส่งผ่านสายแลนผ่านพอร์ตเชื่อมต่อ RJ-45

4.2 สายสัญญาณคอมพิวเตอร์และสื่อกลาง (Network Cables & Transmission Media)
สื่อกลางประเภทสายที่นิยมใช้งานมากที่สุดในระบบเครือข่ายท้องถิ่นคือ สายคู่บิดเกลียว (Twisted Pair Cable) ซึ่งการบิดเกลียวของสายทองแดงจะช่วยลดสัญญาณรบกวน (Crosstalk) โดยแบ่งออกเป็น 2 ประเภทหลัก:

สาย UTP (Unshielded Twisted Pair): สายแลนทั่วไปที่ไม่มีเกราะป้องกันคลื่นแม่เหล็กไฟฟ้า มีราคาประหยัด ยืดหยุ่นสูง นิยมใช้เดินสายภายในบ้านและอาคารสำนักงานทั่วไป

สาย STP (Shielded Twisted Pair): สายแลนที่มีเกราะฟอยล์โลหะหุ้มรอบคู่สายทองแดง เพื่อป้องกันคลื่นรบกวนจากภายนอก เหมาะกับการเดินสายผ่านพื้นที่โรงงานอุตสาหกรรม หรือบริเวณที่มีคลื่นสัญญาณรบกวนสูง

มาตรฐานความเร็วของสายแลนแต่ละระดับ (Category):

CAT 5e: รองรับแบนด์วิดท์ 100 MHz ความเร็วรับส่งข้อมูลสูงสุด 1 Gbps (1,000 Mbps) เหมาะกับระบบเน็ตเวิร์กทั่วไป

CAT 6: รองรับแบนด์วิดท์ 250 MHz ความเร็วสูงสุด 10 Gbps (ในระยะไม่เกิน 55 เมตร) และ 1 Gbps (ที่ระยะ 100 เมตร)

CAT 6A: รองรับแบนด์วิดท์ 500 MHz ความเร็วสูงสุด 10 Gbps เต็มพิกัดได้ไกลถึง 100 เมตร และป้องกันสัญญาณรบกวนได้ดีขึ้นมาก

4.3 มาตรฐานการจัดสีและการเข้าหัวสาย LAN (Wiring Standards)
มาตรฐานในการเรียงลำดับสีทองแดงทั้ง 8 เส้นภายในหัว RJ-45 มี 2 มาตรฐานหลักที่กำหนดโดย TIA/EIA:

มาตรฐาน T568A (มักใช้ในหน่วยงานราชการบางแห่ง):

เรียงลำดับสีดังนี้: 1. ขาว-เขียว / 2. เขียว / 3. ขาว-ส้ม / 4. น้ำเงิน / 5. ขาว-น้ำเงิน / 6. ส้ม / 7. ขาว-น้ำตาล / 8. น้ำตาล

มาตรฐาน T568B (นิยมใช้งานมากที่สุดทั่วโลกเป็นมาตรฐานหลัก):

เรียงลำดับสีดังนี้: 1. ขาว-ส้ม / 2. ส้ม / 3. ขาว-เขียว / 4. น้ำเงิน / 5. ขาว-น้ำเงิน / 6. เขียว / 7. ขาว-น้ำตาล / 8. น้ำตาล

รูปแบบการนำไปใช้งานเชื่อมต่อ:

สายตรง (Straight-through Cable): เข้าหัวแบบ T568B ทั้งสองฝั่ง ใช้เชื่อมต่ออุปกรณ์ต่างชนิดกัน เช่น คอมพิวเตอร์กับสวิตช์ หรือคอมพิวเตอร์เราเตอร์

สายไขว้ (Crossover Cable): เข้าหัวแบบ T568A ฝั่งหนึ่ง และเข้าหัวแบบ T568B อีกฝั่งหนึ่ง ใช้เชื่อมต่ออุปกรณ์ชนิดเดียวกันโดยตรง เช่น คอมพิวเตอร์เข้ากับคอมพิวเตอร์โดยไม่ผ่านสวิตช์

4.4 ขั้นตอนการปฏิบัติการเข้าหัวสายแลน RJ-45 (Termination Steps)
เพื่อให้การเชื่อมต่อแน่นหนา สัญญาณไม่ขาดหาย และมีความเสถียร ควรปฏิบัติตาม 5 ขั้นตอนนี้อย่างละเอียด:

ปลอกเปลือกฉนวนภายนอก: ใช้คีมหรือใบมีดปลอกสาย ค่อย ๆ ขูดและดึงเปลือกนอก (สีฟ้าหรือสีเทา) ออกยาวประมาณ 2 ถึง 3 เซนติเมตร โดยระวังไม่ให้ใบมีดบาดโดนฉนวนหุ้มสายทองแดงสีข้างในขาด

คลี่เกลียวและเรียงสีตามมาตรฐาน: คลายเกลียวสายทั้ง 4 คู่ ดึงสายทองแดงทุกเส้นให้ตรง จากนั้นสลับเรียงลำดับสีตามมาตรฐาน T568B (ขาวส้ม, ส้ม, ขาวเขียว, น้ำเงิน, ขาวน้ำเงิน, เขียว, ขาวน้ำตาล, น้ำตาล) จากซ้ายไปขวา

ตัดสายไฟส่วนเกิน: รวบแนวสายไฟที่เรียงเสร็จแล้วให้เสมอกันและแบนราบ ใช้ใบมีดตัดที่ด้ามคีมตัดปลายสายทองแดงออกเป็นแนวตรงขนานกัน ให้คงเหลือความยาวสายประมาณ 1.5 เซนติเมตร

สอดเข้าหัว RJ-45 และใช้คีมบีบย้ำ: หันหน้าสัมผัสทองแดงของหัว RJ-45 ขึ้น (เอาด้านสลักล็อกลงข้างล่าง) สอดสายแลนเข้าไปจนลึกสุดหน้าสัมผัส โดยตรวจดูว่าปลายเปลือกสายรอบนอกมุดเลยเข้าไปข้างในหัวเพื่อยึดล็อก จากนั้นนำเข้าช่องคีมและออกแรงบีบเพื่อย้ำสลักทองแดงให้กดทับสาย

ทดสอบสัญญาณด้วยเครื่องสแกน (Tester): นำปลายสายทั้ง 2 ด้านเสียบเข้าที่ช่องเครื่องทดสอบสัญญาณ เปิดเครื่องและสังเกตไฟวิ่ง LED ตั้งแต่หมายเลข 1 ถึง 8 หากไฟกะพริบเรียงตรงกันจากบนลงล่างทั้งฝั่ง Master และ Remote แสดงว่าเข้าสายสำเร็จพร้อมใช้งานจริงครับ

## 🌐 บทที่ 5: โปรโตคอล IP Address และการแชร์ทรัพยากร (Protocols & Resource Sharing)

- 5.1 มาตรฐานการสื่อสาร: แบบจำลอง OSI Model 7 ชั้น และชุดโปรโตคอล TCP/IP
- 5.2 โครงสร้าง IP Address (IPv4): ความหมายของ Network ID/Host ID, คลาสไอพี (Class A,B,C), Public/Private IP และ Subnet Mask
- 5.3 การตั้งค่า IP และ Subnetting เบื้องต้น: การกำหนด Static/Dynamic IP และการแบ่งวงเครือข่ายย่อย
- 5.4 การทดสอบและแชร์ทรัพยากร: การใช้คำสั่ง Ping, Tracert ตรวจสอบเครือข่าย และการตั้งค่าแชร์ไฟล์/ปริ้นเตอร์ในวงแลน
  🛠️ กิจกรรมปฏิบัติการ: ออกแบบหมายเลขไอพี ตั้งค่าเชื่อมต่อเครือข่าย และแชร์ข้อมูล (Folder Sharing)

## 🧰 บทที่ 6: การบำรุงรักษาระบบและโปรแกรมอรรถประโยชน์ (System Maintenance & Built-in Utilities)

- 6.1 เครื่องมือจัดการระบบจาก Windows (Built-in Tools): การใช้ Disk Cleanup, Storage Sense และการจัดระเบียบข้อมูล (Defragmentation)
- 6.2 การตรวจสอบและซ่อมแซมไฟล์ระบบ: การใช้งานคำสั่ง Command Prompt พื้นฐาน (sfc /scannow, chkdsk)
- 6.3 การป้องกันและสำรองข้อมูล: ภัยคุกคามไซเบอร์เบื้องต้น, Antivirus, และการสร้าง System Restore Point
- 6.4 การตรวจสอบประสิทธิภาพ (Monitoring Tools): การใช้งาน Task Manager และ Resource Monitor เพื่อหาสาเหตุเครื่องช้า

🛠️ กิจกรรมปฏิบัติการ: วิเคราะห์ประสิทธิภาพ ล้างไฟล์ขยะ ตรวจสอบระบบไฟล์ และจำลองการกู้คืนระบบ (System Restore)

## 🕵️‍♂️ บทที่ 7: การวิเคราะห์และแก้ไขปัญหาไอที (Troubleshooting & Reporting)

- 7.1 ระเบียบวิธีแก้ปัญหา (Troubleshooting Methodology): กระบวนการ 6 ขั้นตอนในการค้นหาสาเหตุและแก้ไขปัญหา
- 7.2 การวิเคราะห์ปัญหาเครือข่ายและฮาร์ดแวร์: การแยกแยะสาเหตุ (Hardware/Software) และการใช้คำสั่งช่วยวิเคราะห์ (ipconfig, netstat)
- 7.3 การอ่านบันทึกระบบ (Windows Logs): การใช้งาน Event Viewer เพื่อสืบสวนข้อผิดพลาดและอาการค้าง/ดับ
- 7.4 การจัดทำระบบเอกสาร (IT Helpdesk Documentation): ความสำคัญของประวัติการซ่อม และการเขียนรายงานอย่างมืออาชีพ
  🛠️ กิจกรรมปฏิบัติการ: จำลองสถานการณ์แก้ปัญหา (Role-play) วินิจฉัยข้อผิดพลาดด้วยเครื่องมือ Windows และจัดทำรายงานแจ้งซ่อม

# การออกแบบส่วนติดต่อผู้ใช้ขั้นสูง (Advanced User Interface Design: 31901-2001)

แผนโครงสร้างหน่วยการเรียนรู้: การออกแบบ UX/UI สำหรับการพัฒนาระบบสารสนเทศ

หลักสูตรเน้นการปฏิบัติจริงแบบ Project-Based Learning (ระยะเวลา 18 สัปดาห์)

## Unit 1: พื้นฐาน UX/UI

- 1.1 นิยามและความแตกต่างของ UX และ UI
  - 1.1.1 นิยามของ User Experience (UX) ในระบบดิจิทัล
  - 1.1.2 นิยามของ User Interface (UI) และ visual elements
- 1.2 โครงสร้างหน้าตาแอปพลิเคชันและหน้าเว็บเบื้องต้น
  - 1.2.1 ส่วนประกอบหลักบนอินเตอร์เฟส (Header, Body, Navigation, CTA Buttons)
  - 1.2.2 การสำรวจและการแยกแยะประเภทของส่วนติดต่อผู้ใช้ในชีวิตประจำวัน
- 1.3 การใช้งาน google Stitch ร่วมกับ design.md
  - 1.3.1 เริ่มต้นใช้งาน google Stitch
  - 1.3.2 การใช้งาน google Stitch ร่วมกับ design.md
- 1.4 การเขียน design.md
  - 1.4.1 โครงสร้างพื้นฐาน ไวยากรณ์ และกฎเกณฑ์ของไฟล์ design.md สำหรับระบุสเปกการออกแบบ
  - 1.4.2 การเขียนรายละเอียดองค์ประกอบ UI และการควบคุมดีไซน์ผ่านไฟล์ Markdown

ใบงานปฏิบัติ 1: ฝึกวิเคราะห์องค์ประกอบ UI ของแอปพลิเคชันยอดนิยมในตลาด, เขียนเอกสารข้อกำหนดการออกแบบเบื้องต้นในรูปแบบ design.md และทดลองใช้ google Stitch ร่วมกับไฟล์ design.md เพื่อสร้างหน้าเว็บจากโจทย์ตัวอย่าง จำนวน 1 ชุด

## Unit 2: การเขียนเอกสารข้อกำหนดการออกแบบ (Design Specification)

- 2.1 เส้นทางเดินของผู้ใช้งานระบบ (User Flow)
  - 2.1.1 ความหมายและความเข้าใจเส้นทางเดินของผู้ใช้งานระบบ
  - 2.1.2 การกำหนดสัญลักษณ์และโครงสร้างแผนผังแสดงทิศทางการทำงานของระบบสารสนเทศ
- 2.2 ตรรกะเงื่อนไขการทำงานในระบบ (Logic Link)
  - 2.2.1 การวิเคราะห์และการเชื่อมโยงตรรกะเงื่อนไขการทำงานในระบบ
- 2.3 โครงสร้างและองค์ประกอบหน้าจออย่างละเอียด
  - 2.3.1 วิธีการระบุโครงสร้างและองค์ประกอบปุ่มในแต่ละหน้าจออย่างละเอียด
- 2.4 การเขียนข้อกำหนดการทำงาน (Specification)
  - 2.4.1 รูปแบบและหลักการเขียนข้อกำหนดการทำงานในรูปแบบไฟล์ Markdown
  - 2.4.2 เทคนิคการแจกแจงหน้าที่ทำงานของหน้าจอบนอุปกรณ์ประเภทต่าง ๆ ด้วยไฟล์ design.md

ใบงานปฏิบัติ 2: เขียนแผนผังแสดงทิศทางการทำงานของแอปพลิเคชันอย่างง่าย และจัดทำไฟล์เอกสารข้อกำหนดระบบ design.md ของโครงงานตนเอง (ประกอบด้วย User Flow, รายการหน้าจอที่ต้องการ และหน้าที่ของปุ่มควบคุมหลัก)

## Unit 3: การสร้างแอปพลิเคชันจำลองระดับสูงด้วย Figma

- 3.1 พื้นฐานการใช้งาน Figma
  - 3.1.1 ส่วนติดต่อผู้ใช้งาน (Figma Interface) และชุดเครื่องมือวาดรูปทรงเบื้องต้น
  - 3.1.2 โครงสร้างและการทำความเข้าใจการทำงานของ Frame และ Group
- 3.2 ระบบ Auto Layout
  - 3.2.1 แนวคิดและระบบ Auto Layout เพื่อจัดการโครงร่างยืดหยุ่นอัตโนมัติ
- 3.3 โครงร่างแบบหยาบ (Lo-Fi Wireframing)
  - 3.3.1 หลักเกณฑ์และเหตุผลในการสร้างโครงร่างแบบหยาบ
  - 3.3.2 วิธีการเปลี่ยนคำอธิบายตัวอักษรในไฟล์ design.md เป็นแบบจำลองแบบไม่มีสีสัน
- 3.4 สถาปัตยกรรมข้อมูล (Information Architecture)
  - 3.4.1 การวิเคราะห์โครงสร้างข้อมูลและการจัดลำดับตารางการนำเสนอบนโครงร่างแบบหยาบ
- 3.5 ทฤษฎีและจิตวิทยาของสีและตัวอักษร
  - 3.5.1 ทฤษฎีและจิตวิทยาการเลือกใช้ระบบสี (Color Schemes & Psychology)
  - 3.5.2 การกำหนดขนาด สไตล์ และน้ำหนักของตัวอักษร (Typography & Typography Hierarchy)
- 3.6 ลำดับความสำคัญทางสายตา (Visual Hierarchy)
  - 3.6.1 หลักการจัดน้ำหนัก ลำดับความสำคัญ และความสมดุลทางสายตาบนหน้าจอแอปพลิเคชันและเว็บ
- 3.7 แบบจำลองระบบโต้ตอบ (Interactive Prototyping)
  - 3.7.1 หลักการทำงานของแบบจำลองระบบโต้ตอบใน Figma
  - 3.7.2 รูปแบบประเภทการเชื่อมต่อหน้าจอ (Trigger, Action & Transition)
- 3.8 สถานะการปฏิสัมพันธ์ (Interactive States)
  - 3.8.1 การตั้งค่าปุ่มและองค์ประกอบย่อยให้เปลี่ยนสถานะได้ (Hover, Pressed, Focused)

ใบงานปฏิบัติ 3: ฝึกวาดรูปทรงและจัดหน้า Layout ด้วย Auto Layout, สร้าง Lo-Fi Wireframe จากไฟล์ design.md, ลงสีใส่ฟอนต์เพื่อพัฒนาเป็น Hi-Fi UI และพัฒนาแบบจำลองโต้ตอบ (Interactive Prototype v.1) ที่กดลิงก์เชื่อมโยงปุ่มสลับหน้าจอไปมาได้เสมือนแอปจริง

## Unit 4: การนำเสนอโครงการกลางภาค (Midterm Project Pitching)

- 4.1 การนำเสนอผลงานจำลองเบื้องต้น
  - 4.1.1 การสรุปประเด็นแนวคิดการออกแบบและสไลด์ประกอบโครงการ

ใบงานปฏิบัติ 4: เข้าร่วมกิจกรรมนำเสนอต้นแบบระบบ (Figma Prototype v.1) และแผนฟังก์ชันที่แปลงมาจาก design.md เพื่อรับฟังคำติชม และจัดส่งสไลด์นำเสนอแนวคิดประกอบลิงก์ตัวจำลองหน้าจอแรกที่กดเล่นได้จริง

## Unit 5: การวิเคราะห์ความต้องการและการคิดเชิงออกแบบ (Design Thinking)

- 5.1 กระบวนการออกแบบโดยใช้ผู้ใช้เป็นศูนย์กลาง (Human-Centered Design)
  - 5.1.1 ภาพรวมและเป้าหมายของกระบวนการออกแบบโดยผู้ใช้เป็นศูนย์กลาง
  - 5.1.2 ขั้นตอน Empathize ของกระบวนการ Design Thinking
- 5.2 เทคนิคสัมภาษณ์เพื่อเข้าใจความต้องการผู้ใช้ (Empathy Interview)
  - 5.2.1 แนวทางการตั้งคำถามและการสังเกตพฤติกรรมผู้ใช้ผ่าน Empathy Interview
- 5.3 การถอดวิเคราะห์ข้อมูลการใช้งาน
  - 5.3.1 วิธีการถอดข้อมูลจากการสัมภาษณ์เพื่อระบุปัญหาและจุดติดขัด (Pain Points) ของผู้ใช้งาน
- 5.4 การสร้างตัวละครผู้ใช้จำลองและแผนภาพความรู้สึก
  - 5.4.1 การสร้างตัวละครผู้ใช้สมมติในรูปแบบการ์ดจำลองพฤติกรรม (User Persona Card)
  - 5.4.2 การจัดระเบียบข้อมูลทางอารมณ์และพฤติกรรมด้วยตาราง Empathy Map
- 5.5 การคิดค้นฟังก์ชันการแก้ไขปัญหา (Ideation)
  - 5.5.1 ขั้นตอนและเทคนิคการระดมสมองเพื่อแก้ปัญหาความต้องการ (Ideation Techniques)
  - 5.5.2 การวิเคราะห์จัดเรียงความสำคัญของคุณสมบัติการทำงาน (Feature Prioritization)
- 5.6 การทำแผนภาพความต้องการระบบ (Requirement Map)
  - 5.6.1 การวาดแผนภาพความต้องการเชิงระบบ (User Requirement Mapping) เพื่อตอบปัญหาจาก Persona

ใบงานปฏิบัติ 5: ดำเนินกิจกรรมสัมภาษณ์เพื่อเก็บข้อมูลและระบุข้อจำกัดในการใช้งาน Prototype ของเพื่อน, สังเคราะห์ข้อมูลเพื่อสร้าง User Persona Card, Empathy Map และ User Requirement Map สำหรับกำหนดฟังก์ชันที่ตอบสนองความต้องการผู้ใช้จริง

## Unit 6: จิตวิทยาการออกแบบและความง่ายในการใช้งาน (Usability & Psychology)

- 6.1 จิตวิทยาเกสตัลท์สำหรับการออกแบบอินเตอร์เฟส
  - 6.1.1 จิตวิทยาการรับรู้ของมนุษย์ผ่านกฎเกณฑ์การจัดกลุ่มทางทัศนศิลป์ (Gestalt Principles)
- 6.2 การลดภาระการประมวลผลและการใช้กฎของฮิก (Hick's Law)
  - 6.2.1 วิธีการลดภาระในการประมวลผลข้อมูลและความทรงจำบนหน้าจอ (Cognitive Load Reduction)
  - 6.2.2 การประยุกต์ใช้กฎของฮิก (Hick's Law) ในการปรับโครงสร้างหน้าจอและจำกัดตัวเลือกที่สับสน
- 6.3 การประเมิน Usability Heuristics เบื้องต้น
  - 6.3.1 แนวคิดและหลักประเมินการออกแบบส่วนติดต่อผู้ใช้ที่ใช้งานง่าย (Usability Heuristics 10 ข้อเบื้องต้น)
- 6.4 การตอบกลับสถานะการทำงาน (Visual Feedback)
  - 6.4.1 ความสำคัญและการใช้สี/ภาพตอบสนองสถานะการกดสั่งการของระบบ (Visual Feedback & System Status)
  - 6.4.2 การขัดเกลาลำดับสายตา (Visual Hierarchy Clean-up) บนส่วนติดต่อผู้ใช้ระดับสูง

ใบงานปฏิบัติ 6: ดำเนินกิจกรรมวิเคราะห์และทำ UI Makeover ปรับปุ่มและองค์ประกอบที่ซ้ำซ้อนออกตามกฎของฮิก, ขัดเกลาลำดับสายตา (Visual Hierarchy) และพัฒนาไฟล์ออกแบบใน Figma เวอร์ชั่นปรับปรุงใหม่ (Refined UI Design v.2) ที่มีสถานะปุ่มและการตอบสนองตามจิตวิทยาการออกแบบครบถ้วน

## Unit 7: การทดสอบและการปรับปรุงแก้ไข (Testing & Iteration)

- 7.1 การเตรียมการทดสอบการใช้งานระบบ
  - 7.1.1 จุดประสงค์ของกระบวนการทำ Usability Testing (UT) แบบมีแบบแผน
  - 7.1.2 การเขียนสร้างสถานการณ์และโจทย์ภารกิจสมมติในการใช้งานระบบ (Task Scenario Design)
- 7.2 บทบาทผู้ดำเนินการทดสอบ
  - 7.2.1 บทบาทสำคัญและมารยาทที่ควรมีสำหรับผู้เป็นผู้สังเกตการณ์การทดสอบระบบ
- 7.3 กระบวนการทดสอบผู้ใช้งาน (UT Session)
  - 7.3.1 ขั้นตอนดำเนินกิจกรรมการทดสอบระบบกับผู้ใช้งาน (UT Session Management)
- 7.4 เทคนิคเก็บสังเกตการณ์และสัมภาษณ์สรุปผล
  - 7.4.1 เทคนิคการบันทึกข้อมูลพฤติกรรมและการตรวจจับความสับสนของผู้ใช้ (Observation Techniques)
  - 7.4.2 การสัมภาษณ์ความรู้สึกภายหลังจากการทดสอบและจัดกลุ่มข้อมูลข้อบกพร่อง
- 7.5 การเคลียร์ไฟล์ดีไซน์ (Figma Clean-up)
  - 7.5.1 การจัดระเบียบโครงสร้าง เลเยอร์ และชื่อหน้าจอภายใน Figma ให้เป็นมาตรฐานสากล
- 7.6 โหมดนักพัฒนาและการเตรียมส่งออกทรัพยากร (Dev Mode & Assets Export)
  - 7.6.1 การใช้เทคโนโลยีและโหมดนักพัฒนา in Figma (Dev Mode & Spec Inspection)
  - 7.6.2 ขั้นตอนและแนวทางการเตรียมทรัพยากรดีไซน์ (Exporting Assets เช่น ไอคอน, รูปภาพ) ส่งต่อโปรแกรมเมอร์

ใบงานปฏิบัติ 7: ออกแบบรายการภารกิจการใช้งาน 3-5 ข้อ, สลับบทบาทเพื่อดำเนินการทำ Usability Testing กับผู้ใช้จริงและจดบันทึกพฤติกรรมความสับสน, จัดทำสรุปรายงาน Usability Testing Report ขนาด 1 หน้ากระดาษ, และจัดระเบียบองค์ประกอบใน Figma ส่งมอบงานแบบสมบูรณ์ (Final Design Deliverable)

## Unit 8: โครงงานและการนำเสนอชิ้นงานปลายภาค (Final Project & Presentation Showcase)

ใบงานปฏิบัติ 8: จัดทำกรณีศึกษาผลงานออกแบบ (UX/UI Case Study) ของตนเอง และนำเสนอแนวคิดการออกแบบ กระบวนการตั้งแต่ทำความเข้าใจผู้ใช้ ไปจนถึงตัวต้นแบบแอปพลิเคชันที่สมบูรณ์ผ่านกิจกรรมจัดแสดงผลงาน (Showcase Portfolio)

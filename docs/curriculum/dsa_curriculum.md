## Unit 2: Stack และ Queue

- 2.1 ทฤษฎีและแนวคิดของโครงสร้างข้อมูลแบบ Stack
  - 2.1.1 นิยามและหลักการทำงานแบบ LIFO (Last In, First Out)
  - 2.1.2 โอเปอเรชันพื้นฐานของ Stack (Push, Pop, Peek/Top, isEmpty และ Size)
- 2.2 การสร้างและใช้งาน Stack ในภาษา Python
  - 2.2.1 การใช้งานแบบพื้นฐานด้วยโครงสร้าง List
  - 2.2.2 การเพิ่มประสิทธิภาพในการเขียนโปรแกรมด้วยโมดูล collections.deque
- 2.3 การประยุกต์ใช้งาน Stack ในระบบจริง
  - 2.3.1 การตรวจสอบความถูกต้องของการจับคู่วงเล็บ (Balanced Parentheses)
  - 2.3.2 บทบาทของ Stack ในการเรียกใช้ฟังก์ชัน (Call Stack) และการทำงานแบบ Recursion
- 2.4 ทฤษฎีและแนวคิดของโครงสร้างข้อมูลแบบ Queue
  - 2.4.1 นิยามและหลักการทำงานแบบ FIFO (First In, First Out)
  - 2.4.2 โอเปอเรชันพื้นฐานของ Queue (Enqueue, Dequeue, Front, Rear, isEmpty และ Size)
- 2.5 การสร้างและใช้งาน Queue ในภาษา Python
  - 2.5.1 ข้อจำกัดและผลกระทบด้านประสิทธิภาพหากใช้ List ทั่วไปในการสร้าง Queue
  - 2.5.2 การสร้าง Queue อย่างมีประสิทธิภาพด้วยโมดูล collections.deque
- 2.6 การประยุกต์ใช้งาน Queue ในระบบจริง
  - 2.6.1 การจัดการคิวงานในระบบปฏิบัติการ (Task Scheduling และ Print Spooling)
  - 2.6.2 การประยุกต์ใช้ในอัลกอริทึมการค้นหา (Breadth-First Search)

## Unit 3: โครงสร้างข้อมูลแบบต้นไม้ (Tree)

- 3.1 ความรู้เบื้องต้นเกี่ยวกับโครงสร้างข้อมูลแบบต้นไม้
  - 3.1.1 Tree คืออะไรและทำไมจึงจัดเป็นโครงสร้างข้อมูลแบบไม่เป็นเส้นตรง (Non-Linear Data Structure)
  - 3.1.2 โครงสร้างความสัมพันธ์แบบลำดับขั้น (Hierarchical Relationship) ในระบบคอมพิวเตอร์ (เช่น ระบบโฟลเดอร์ในคอมพิวเตอร์ หรือโครงสร้าง DOM ใน HTML)
  - 3.1.3 คำศัพท์และส่วนประกอบสำคัญของต้นไม้ เช่น Node, Root, Parent, Child, Leaf Node, Edge, Height และ Depth
- 3.2 โครงสร้างต้นไม้แบบไบนารี (Binary Tree)
  - 3.2.1 Binary Tree คืออะไรและคุณสมบัติพื้นฐาน (มีลูกได้ไม่เกิน 2 โนด)
  - 3.2.2 ความแตกต่างระหว่าง Binary Tree ทั่วไป และ Binary Search Tree (BST)
  - 3.2.3 ข้อดีของ Binary Search Tree ในด้านการเพิ่มความเร็วในการค้นหา จัดเก็บ และจัดเรียงข้อมูล
  - 3.2.4 ข้อเสียและข้อจำกัดของ Binary Tree เช่น ปัญหาต้นไม้เอียง (Skewed Tree) ที่ทำให้ประสิทธิภาพลดลงเหลือเท่ากับ List
- 3.3 วิธีการท่องเข้าไปในต้นไม้ (Tree Traversal)
  - 3.3.1 แนวคิดการท่องเข้าไปในต้นไม้แบบลึกก่อน (Depth-First Search: DFS)
  - 3.3.2 การท่องแบบตามลำดับ (In-order Traversal: ซ้าย-ราก-ขวา) และการนำไปใช้
  - 3.3.3 การท่องแบบก่อนลำดับ (Pre-order Traversal: ราก-ซ้าย-ขวา) และการนำไปใช้
  - 3.3.4 การท่องแบบหลังลำดับ (Post-order Traversal: ซ้าย-ขวา-ราก) และการนำไปใช้
